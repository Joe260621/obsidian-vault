## 一句话总结

**AI Agent = 能查资料(RAG) + 能动手干活(Tool Calling) + 能思考循环(ReAct)。理解了这三件事，就理解了AI在运营中的技术边界——知道什么能做、什么不能做、什么时候该转人工。**

---

## RAG——检索增强生成

### 运营能听懂的解释
> RAG就是给AI配了一本可以实时查阅的产品手册——不靠记忆，靠翻书。

### 运营场景举例
用户在企微群问"健身卡多少钱"，AI不是凭记忆瞎编，而是先检索酒店产品知识库，找到健身卡的价格信息，再基于检索结果回答。

### 运营视角下的关键理解
- **能做**：让AI基于公司自有资料回答——产品信息、FAQ、SOP、会员权益
- **不能做**：知识库里没有的，AI查不到。不是AI笨，是没给它资料
- **运营该关心的**：知识库包含什么？谁来更新？怎么确保信息不过期？

---

## Tool Calling——AI自己调用工具

### 运营能听懂的解释
> Tool Calling就是给AI配了一个工具箱——需要查价格就自己拿价格工具，需要查库存就自己拿库存工具，不用每次都问人。

### 运营场景举例
用户在群里问"自助晚餐券还有货吗"，AI自动调用 check_stock 函数 → 返回"库存充足，剩余200件"。

### 运营视角下的关键理解
- **能做**：让AI自动查数据库、发消息、创工单、下单——不只是聊天，还能帮用户完成操作
- **不能做**：复杂业务逻辑（如"库存不足时自动从其他渠道调货"），那需要多个工具配合
- **运营该关心的**：给AI开放哪些能力？边界在哪？出错了怎么降级（而不是让客户看到报错）？

---

## ReAct——Agent的思考循环

### 运营能听懂的解释
> ReAct是AI的"想一想再做，做完再想"——像人解决问题一样，走一步看一步。

### 运营场景举例
用户在群里说"帮我查自助晚餐还有没有，有的话帮我下单2份"。AI的思考过程：

```
第1步 想想：用户要查库存 → 行动：查库存 → 发现：200件充足
第2步 想想：库存够了，但还要知道价格 → 行动：查价格 → 发现：¥208/位
第3步 想想：信息齐了 → 回复："库存充足，¥208/位，2份共¥416。确认下单吗？"
```

### 运营视角下的关键理解
- **能做**：AI自动判断"信息还不够"，继续查，直到信息齐全
- **不能做**：无限循环。需要设置上限（如5步），超出自动转人工
- **运营该关心的**：循环步数上限、每步耗时监控、转人工的上下文传递是否完整

## 三项技术的运营价值排序

| 技术 | 运营场景价值 | 推荐学习优先级 |
|------|-----------|-------------|
| RAG | ⭐⭐⭐⭐⭐ | 最实用——知识库问答、FAQ自动化、产品信息查询 |
| Tool Calling | ⭐⭐⭐⭐ | 进阶——自动查库存/价格、自动建工单、自动发消息 |
| ReAct | ⭐⭐⭐ | 高级——复杂多步骤任务（下单+查库存+确认+登记一条龙） |

## 面试时怎么聊这段

> 我对AI在运营中的应用，不只是停留在"用ChatGPT写文案"这个层次。我系统学习过RAG（检索增强生成）、Tool Calling（工具调用）、ReAct（思考循环）三个核心概念，而且用Python跑过Demo——把希尔顿76款产品做成知识库，AI能自动查价格、查库存。这意味着我在评估一个AI运营工具的时候，能判断它"技术上能做到什么、边界在哪"，不会被厂商忽悠。
