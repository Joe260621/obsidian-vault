## 项目来源
在希尔顿做用户运营时，发现企微群客户咨询回复太慢（平均30分钟+），部分客户等不到回复就流失了。这是真实的业务痛点。

## 项目性质
个人产品实践项目。2026年6月完成，用时3天。
独立完成：需求分析 → PRD → 原型 → AI Demo验证 → 架构设计。

## 项目产出

### 1. PRD文档
- 用户故事：作为一名酒店客户，我想在企微群咨询时得到即时回复
- 功能需求：自动识别意图 / 查知识库回复 / 查库存代预约 / 转人工
- 非功能需求：回复<30秒 / 意图识别>90% / 预约信息准确率>95%
- 验收标准：4条可量化标准

### 2. Figma原型
- 手机端对话界面
- 用户绿色气泡 + AI灰色气泡 + 底部输入框
- 展示了"自助晚餐怎么预约 → AI自动回复"的完整对话流

### 3. AI Demo（核心亮点）
- 把希尔顿76款真实产品数据做成RAG知识库
- AI能基于知识库回答产品问题（非胡编）
- 接入了Tool Calling：AI自动查价格、查库存
- 方案验证完成——证明了技术可行性

### 4. 架构设计
- 4层架构：意图识别 → 路由分发 → 工具执行 → 回复降级
- 定义了每个AI能力的边界（什么AI做、什么转人工）
- 定义了5个核心指标（意图识别率/回复准确率/任务完成率/转人工率/响应时间）

### 5. 指标体系
| 指标 | 目标 | 说明 |
|------|------|------|
| 意图识别率 | >90% | 能否听懂用户想干什么 |
| 回复准确率 | >85% | AI的回答对不对 |
| 任务完成率 | >70% | 不需要转人工的占比 |
| 转人工率 | <30% | 多少问题AI搞不定 |
| 响应时间 | <30秒 | 从问到AI回复的时间 |
