# 看图 Skill — 识别图片文字 + 智能分析

## 触发词
- "看图"
- "识别图片" / "OCR"
- "图片里写了什么" / "帮我看看这张图"
- "/看图"

## 执行流程

### Step 1: 提取剪切板图片文字
```bash
node "D:/ProjectManagement/Claude/scripts/ocr-clipboard.js"
```

引擎选择规则：
- 用户说"用千问看图" → 追加 `--qwen`（需 QWEN_API_KEY 环境变量）
- 用户说"用 kimi 看图" → 追加 `--kimi`（需 KIMI_API_KEY 环境变量）
- 默认 → tesseract.js（免费离线，中英文混排）

### Step 2: 分析 OCR 结果
OCR 返回文字后，根据用户问题分析：
- 没具体问题 → 用一句话总结图片内容
- 有具体问题 → 基于 OCR 文字回答问题，引用原文

### 提示词模板
```
用户截图 OCR 结果如下：
---
<OCR_TEXT>
---

用户问题：<USER_QUESTION>

请基于以上文字内容回答。如果 OCR 结果不完整或有明显错误，请说明。
```

## 引擎对比
| 引擎 | 费用 | 准确率 | 适用场景 |
|------|:--:|:--:|------|
| tesseract.js | 免费 | 中等 | 简单排版、离线环境 |
| Qwen VL | ~0.001元/张 | 高 | 中英文混排、复杂排版 |
| Kimi | 按量计费 | 高 | 备选方案 |

## 常见问题
| 问题 | 解决 |
|------|------|
| "剪切板中没有图片" | 先截图（Win+Shift+S），确保图片在剪切板 |
| tesseract.js 中文乱码 | 首次运行会下载中文语言包，需联网；或改用 Qwen VL |
| macOS 剪切板读不到 | 安装 pngpaste: `brew install pngpaste` |
| 图片太大识别慢 | tesseract.js 处理 4K 截图约 5-10 秒，正常 |
