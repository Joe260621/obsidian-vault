## 一、底层逻辑：B2B裂变的"三个不等于"

### 1.1 B端裂变 ≠ C端裂变

| 维度 | C端裂变（拼多多/瑞幸） | B端裂变（一手APP） | **对活动设计的影响** |
|:-----|:--------|:----------------|:------------------|
| **驱动力** | 补贴刺激、社交货币、娱乐性 | 同行信任、真实收益、供应链价值 | B端不能靠"红包裂变"，要靠"信任背书" |
| **决策成本** | 低（几块钱的券就下单） | 高（推荐错了丢的是生意面子） | 首单保障机制是B端裂变的**必选项** |
| **传播路径** | 弱连接（朋友圈/群广撒网） | 强连接（同行间的精准推荐） | 裂变设计要服务"一对一推荐"场景 |
| **K因子** | 可做到1.5-3 | 天然0.3-0.8（但LTV高5-10倍） | 不追求病毒式，追求**精准高价值** |
| **羊毛党** | 大量（手机号批量注册） | 极少（虚假店主难获利） | B端风控侧重**关系异常**不是**设备异常** |

**一句话：B端裂变不是让店主去"拉人"，是让店主去"做推荐"——前者是任务，后者是人情。** 任务靠激励推动，人情靠信任拉动。

### 1.2 B端裂变的核心飞轮

```
     ┌───────────┐
     │ 店主A在   │
     │ 一手拿货  │
     └─────┬─────┘
           ↓
     ┌───────────┐         ┌───────────┐
     │ 内容/交易  │ ─────→  │ 同行店主B  │
     │ 自然产生  │  看到    │ 感兴趣   │
     └───────────┘         └─────┬─────┘
                                 ↓
     ┌───────────┐         ┌───────────┐
     │ A获得奖励  │ ←─────  │ B通过A的   │
     │ +信任背书  │  转化    │ 推荐注册   │
     └───────────┘         └───────────┘
                                 ↓
                          ┌───────────┐
                          │ B也成为   │
                          │ 推荐节点   │ ← 第二圈裂变开始
                          └───────────┘
```

**飞轮加速的三个杠杆：**

| 杠杆 | 作用 | 用量化指标衡量 |
|:-----|:-----|:--------------|
| **信任系数** | 老店主推荐→新店主信任度天然是广告的5-10倍 | 推荐转化率 vs 广告转化率 |
| **利益杠杆** | 现金>积分>权益>荣誉（B端店主对现金敏感） | 参与率/拉新数 |
| **内容杠杆** | 真实交易内容=信任证明（晒单比海报管用） | 内容分享率/带新转化率 |

## 二、一手APP用户画像拆解（细颗粒度到可执行）

> **前提认知**：一手800万服务店主不是"一群人"，是至少5个截然不同的子群体。同一个裂变活动对不同类型的店主，效果可能差10倍。以下按"裂变价值"（推荐意愿×推荐能力）做三级拆解。

### 2.1 第一层：按经营形态拆（最粗但最关键的维度）

| 形态 | 占比（估） | 典型特征 | 对裂变的意义 |
|:-----|:---------:|:---------|:------------|
| **线下实体店主** | ~55% | 街边店/商场店/社区店，女性为主，25-40岁 | 同行社交圈最密集（本地服装商圈），裂变信任度高但触达难度大 |
| **直播电商卖家** | ~20% | 抖音/快手直播卖货，年轻（22-30岁），男女均有 | 互联网原住民，分享意愿高，线上社交活跃——裂变最容易放量 |
| **淘宝/拼多多C店** | ~15% | 纯线上经营，25-35岁 | 社交活跃度中等，但被"一件代发"需求驱动，裂变动机偏向功能性 |
| **档口/批发商** | ~5% | 本身就是上游，有稳定供应链 | 裂变价值极高（社交圈全是同行），但不是一手核心用户群 |
| **其他（微商等）** | ~5% | 朋友圈/社群卖货 | 裂变意愿高但信任背书弱（容易被同行看不起） |

### 2.2 第二层：按地域&城市层级拆（影响裂变方式）

| 城市层级 | 占店主比（估） | 裂变环境特征 | 最适合的裂变方式 |
|:---------|:------------:|:------------|:----------------|
| **一线/新一线**（广深/杭/成等） | ~35% | 店主集中在服装商圈，有线下社交场景（档口/批发市场） | 线下口碑裂变+社群内容裂变 |
| **二线/三线** | ~40% | 分散经营，线上社交为主，同行圈相对松散 | 社群裂变+利益驱动裂变 |
| **县城/乡镇** | ~25% | 本地同行熟人多，社交圈小但深度强 | 信任裂变（熟人口碑）+首单保障 |

**关键洞察**：三四线&县域店主对"同行推荐"的信任度比一线城市高30%+——因为一线城市店主见的平台多、选择多、对"营销套路"免疫度高。**裂变活动优先在非一线城市放量，ROI更高。**

### 2.3 第三层：按代际行为拆（关键差异）

| 维度 | 85后（30-35岁） | 90后（26-30岁） | 95后/00后（22-26岁） |
|:-----|:--------------|:---------------|:-------------------|
| **占比估算** | ~20% | ~45% | ~35% | 
| **开店类型** | 实体店为主 | 实体+线上混合 | 直播/社交电商为主 |
| **选款方式** | 看档口、信熟人推荐 | 线上看款+数据参考 | 信平台算法推荐 |
| **内容习惯** | 图文为主 | 图文+短视频 | 短视频+直播 |
| **社交分享** | 分享意愿中低，怕丢面子 | 分享意愿中高，有选择性 | 分享意愿高，日常习惯 |
| **对奖励敏感度** | 现金>积分>荣誉 | 现金≈荣誉>积分 | 荣誉>现金>积分 |
| **裂变触发点** | "这款我卖了3年" | "同行都在推这个款" | "这个款火了" |
| **信任决策** | 熟人推荐>平台背书 | 平台背书≈熟人推荐 | 内容口碑>熟人推荐 |

**核心结论：90后是当前主力（~45%），95后/00后是裂变最优群体（~35%且快速上升中）。** 裂变活动的语言、视觉、激励方式必须向95后/00后倾斜——短视频化、荣誉驱动、低门槛参与。

### 2.4 第四层：按经营阶段拆（决定裂变动机强弱）

| 阶段 | 开店时长 | 占店主比 | 核心焦虑 | 裂变动机 | 最适合的裂变角色 |
|:-----|:--------|:--------|:---------|:---------|:----------------|
| **探索期** | ＜6个月 | 15% | "不知道选什么款，怕压库存" | 学习大过分享，裂变意愿低 | 裂变受众（被推荐者） |
| **成长期** | 6月-2年 | 35% | "想找到稳定的供应链" | 有分享意愿但怕推荐错了丢面子 | **潜在推荐官**——需要首单保障降低风险 |
| **成熟期** | 2-5年 | 35% | "怎么提高利润、降低进货成本" | 分享意愿最强——已经有同行社交圈，有自己的选款判断 | **裂变主力军**——最适合做推荐官 |
| **衰退期** | 5年+ | 15% | "客流下滑，要不要换行" | 分享意愿下降，经营疲惫 | 被动裂变（被动的信任背书） |

### 2.5 第五层：按"裂变DNA"拆（最关键的颗粒度——谁能裂变、谁不能）

不是所有店主都具备裂变能力。以下六种人格决定裂变效率：

```
                       社交推荐能力
                            │
                            │
     旁观店主 ←────────────┼────────────→  裂变发动机
   (5%, 不参与不分享)       │              (15%, 核心推荐官群)
                            │
                            │
   ─────────────────────────┼────────────────→  社交影响力
                            │
                            │
     内容贡献者 ←──────────┼────────────→  信任枢纽
   (30%, 发内容但不主动拉)    │              (50%, 沉默但可信)
```

| 裂变DNA类型 | 占比 | 特征一句话 | 裂变价值 | 怎么用 |
|:-----------|:---:|:----------|:-------:|:-------|
| **🔥 裂变发动机** | 5-10% | "我来推荐，没问题"——有影响力+有意愿 | ⭐⭐⭐⭐⭐ | 首批推荐官人选，给最高级别激励+专属身份 |
| **🧩 信任枢纽** | 40-50% | "我不主动推，但朋友问我我会说好" | ⭐⭐⭐⭐ | 最大的沉默裂变资产——通过"晒单内容"让他们被看到，间接裂变 |
| **📝 内容贡献者** | 20-30% | "我喜欢晒单和分享选款心得" | ⭐⭐⭐ | 他们的内容就是最好的裂变素材——AI工具帮他们把内容变成分享海报 |
| **👁️ 旁观店主** | 5-10% | "不想参与这些" | ⭐ | 不动他们，不浪费运营资源 |
| **💰 利益猎人** | 10-15% | "有奖励我就来" | ⭐⭐ | 参与活动但不深度绑定，用一次性奖励驱动 |
| **⚠️ 风险用户** | 1-3% | "想薅羊毛/钻空子" | ❌ | 风控重点监控对象 |

### 2.6 三类"高裂变价值"店主的详细人物卡

#### 人物卡①：「直播带货王」小陈

```
年龄：24岁   |   城市：杭州
店铺：抖音女装直播间   |   粉丝：5万
经营阶段：成长期（1.5年）
月采购：¥8,000-15,000（每周上新3-4次）
平台行为：日均打开APP 3-5次，看爆款预测+选款推荐

社交画像：
├ 线上：3个WX同行群（活跃），抖音好友200+同行主播

  ✅ "我给你推荐个靠谱的供应商" → 互助驱动力

首选裂变方式：内容裂变（晒爆款→生成选款海报→发到同行群）
次选：话题裂变（#这款我卖爆了 参与话题）
不选：纯利益驱动的拉新任务
```

## 三、4大能力排序：哪个最重要？为什么？

### 排名：① > ② > ③ > ④

### 🥇 第1：策划并执行裂变活动，设计分享激励玩法（最重要）

**为什么是它？——因为这是"裂变"本身。其他三个都是围绕它服务的。**

在一手这个场景下，这条能力之所以最重要，有三个无法替代的原因：

**原因A：B端裂变的活动设计是最没有"标准答案"的能力**
> ROI复盘可以学，用户行为分析可以练，风控可以抄作业。但**设计一个"店主愿意分享、分享不丢面子、被推荐人信任、同时合规"的B端裂变活动**——这个没有模板，全靠对用户的理解深度。你面对的不是普通消费者，是"推荐错了丢生意面子"的店主。C端的"分享得红包"搬过来就直接踩坑。

**原因B：活动设计决定了其他三个能力的起点**
> 如果活动设计错了——激励方向错了、分享路径卡住了、话术让店主感觉像传销——**你再会分析行为、再会算ROI，也只能在错误的方向上优化**。这就好比你菜炒咸了，再精妙的摆盘也救不回来。
>
> ```
> 活动设计(对) → 行为分析(有用) → ROI优化(有效果)
> 活动设计(错) → 行为分析(浪费) → ROI优化(浪费×2)
> ```

**原因C：谦腾的经历正好证明了这个排序**
> 你在谦腾为什么能跑通？不是因为你会算ROI（那时候你可能还没算那么细），不是因为你会分析行为（数据没那么全），更不是因为风控（恰恰是风险最后搞死了）。**核心是因为你把"花生日记式分销"翻译成了"欧芮儿私域裂变"——活动设计对了，裂变自己跑起来了。**
>
> 同时谦腾的教训也证明了：**活动设计必须包含"风险评估"（不是作为单独的能力，而是作为活动设计的一部分）。** 如果你在活动设计阶段就评估了二级分销的风险，可能不会走到粉丝反噬那一步。

### 🥈 第2：分析用户行为，迭代活动玩法

**为什么第二？——因为你不可能一次做对。裂变是迭代出来的，不是设计出来的。**

即使你的活动设计方向对了，具体的「奖励金额定¥30还是¥50」「海报用什么配色」「话术用"邀请"还是"推荐"」「注册页要不要加手机验证」——这些问题上线的第一版永远不是最优解。

这个能力的价值在于：

```
没有行为分析的活动优化 = 猜
有行为分析的活动优化 = 验证

猜： "我觉得¥50比¥30效果更好"
验证："A组¥30转化率12%，B组¥50转化率14%，差异不显著，加测到样本量1000"
```

你在面试时可以说硕牛的RFM分层经验（"我在硕牛用RFM分层把130万用户分成12个群体，每个群体的活动参与率差异达到3-5倍"）。

### 🥉 第3：复盘ROI，优化投入产出

**为什么第三？——ROI是"验证"能力，不是"创造"能力。它告诉你对不对，但不告诉你什么是对的。**

一个好的运营可以不懂ROI吗？不行。但ROI复盘的价值**依赖于前两个能力已经把事情做对了**。

这个能力在面试时的差异化价值在于：**不是简单地"花了多少钱÷拉了多少人"，而是算"LTV/CAC/回本周期/裂变网络价值"**——用你在万富做数据的功底。

### 第4：监控并规避活动风险（羊毛党）

**为什么第四？——不是不重要，是"可以部分工具化/系统化"。**

B端羊毛党的破坏力比C端小，而且70%的风险可以通过规则层自动拦截、20%通过算法层识别、10%人工处理。这套体系**搭好之后是"维持"不是"创造"**。

**但这里有一个关键区别：** 对于一手这个岗位，**最大的风险不是羊毛党，而是"推荐信任风险"**——推荐人为了拿奖励狂推、被推荐人来了之后体验差、双方都对一手失去信任。这是B端特有的风险，和C端的羊毛党是完全不同层面的问题。

所以第四能力在B端的真正含义不是"防刷单"，是**"防信任崩塌"**。

### 面试时怎么说这个排序（30秒版）：

> "四个能力按重要性排序：**第一是活动设计**——B端裂变最难的是让店主'愿意分享又不丢面子'，活动设计对了其他都有意义；**第二是行为分析**——因为没有人能一次做对，需要通过数据迭代找到最优解；**第三是ROI复盘**——验证方向对不对、值不值得继续投；**第四是风控**——B端最主要的风险不是羊毛党，是推荐太多伤了信任。前两个是创造价值，后两个是守住价值。"

### 能力1：策划并执行裂变活动，设计分享激励玩法

#### 🧠 底层逻辑

B端分享激励的设计公式：

```
分享意愿 = 推荐收益 × 信任背书 - 推荐风险
         ──────────────────────────────
                     决策成本
```

- **推荐收益**：不只是现金，还有行业影响力、社交资本
- **信任背书**：推荐人用自己的信誉做担保，平台要给推荐人"安全感"
- **推荐风险**：推荐错了→朋友亏钱→推荐人丢面子→不可逆
- **决策成本**：B端推荐比C端高10倍，每一步都要降低

#### 🔧 操作思路

**Step 1：选择裂变种子（不是越多越好，是越"敢推荐"越好）**
- 筛选标准：月采购≥3次 + 社群发言≥10次/月 + 朋友圈有选款内容
- 首批只选500-1000人，给专属邀请码+专属海报
- 关键话术：**不是"你来帮我们拉人"，而是"给你一个让同行更认可你的机会"**

**Step 2：设计激励结构（现金打底，荣誉放大）**

```
基础激励（必选）：
  ┌────────────────────────────────────┐
  │ 推荐朋友注册 → 朋友首次下单后 →     │
  │ 推荐人得 ¥30-50 现金               │
  │ 被推荐人得 ¥20 首单券               │
  └────────────────────────────────────┘

进阶激励（可选）：
  ┌────────────────────────────────────┐
  │ 推荐3人 → 解锁"选款达人"认证        │
  │ 推荐10人 → 获"一手推荐官"年度权益    │
  │ 推荐50人 → 加入"店主顾问委员会"      │
  │   （参与选品、获独家供应链信息）      │
  └────────────────────────────────────┘
```

**关键设计细节：**
- 奖励**后置**（朋友真实下单后才到账）→ 防止刷单
- 现金**即时**到账（微信/支付宝）→ 建立信任，形成正反馈
- 荣誉**分层**（从认证到顾问）→ 长期绑定，让推荐成为习惯
- 被推荐人也得有明确收益 → 这是"利他推荐"的底线

**Step 3：设计分享物料（简化到"一键完成"）**
- 店主分享路径不超过2步：点击"邀请同行"→ 立即生成海报
- 海报内容：店主昵称+头像+一句推荐语+一手的爆款截屏+注册二维码
- **海报上不加任何"广告感"元素**，就是"一个店主在分享好货源"的样子

**Step 4：选对活动场景**
- 场景一：店主下单后→弹窗"分享这本款给你的同行朋友"
- 场景二：店主晒单内容发布后→自动生成分享海报
- 场景三：社群内发起"同行拼单"→自动记录邀请关系

### 能力2：分析用户行为，迭代活动玩法

#### 🧠 底层逻辑

裂变是**概率游戏**，不是一次设计就能成功的。核心是做**假设→验证→放大→再假设**的循环。

```
       假设                设计              验证
  ┌────────────┐    ┌────────────┐    ┌────────────┐
  │"¥50比¥30  │ →  │ A组¥50    │ →  │A组拉新率  │
  │ 拉新更高" │    │ B组¥30    │    │B组拉新率  │
  └────────────┘    └────────────┘    └──────┬─────┘
                                             ↓
                                     数据决策
                                 ┌──────────────┐
                                 │ 效果显著？→ 放量 │
                                 │ 效果不显著？→ 换假设│
                                 └──────────────┘
```

#### 🔧 操作思路

**A. 建立裂变漏斗指标（每天盯这4个数）：**

```
                   曝光（海报被看到）
                       ↓
                   点击（扫码打开）
                       ↓
                   注册（被推荐人填手机号）
                       ↓
                   首单（完成第一次采购）
```

| 层级 | 行业参考值 | 对标意义 |
|:-----|:---------:|:---------|
| 曝光→点击率 | 30-50% | 海报/推荐文案是否有吸引力 |
| 点击→注册率 | 60-80% | 注册流程是否顺畅 |
| 注册→首单率 | 30-50% | 首单转化（新用户成长体系是否有效） |
| **整体CAC** | **¥50-100/人** | 包含推荐奖励+物料+运营成本 |

**B. 每次迭代只改一个变量**

| 迭代假设 | 变量 | 对照组 | 实验组 | 观察周期 |
|:---------|:-----|:-------|:-------|:--------|
| "¥50比¥30诱人" | 奖励金额 | ¥30 | ¥50 | 2周 |
| "到账速度影响参与率" | 到账方式 | 次日到账 | 实时到账 | 1周 |
| "选款达人认证更有吸引力" | 激励类型 | 仅现金 | 现金+身份认证 | 2周 |

**C. 用RFM模型筛选最优推荐人**

借鉴你在硕牛的RFM经验：
- **R（最近采购）**：近7天有采购的店主→推荐意愿最高（刚体验过价值）
- **F（采购频次）**：月采购4次+→对平台最认可，推荐内容最真实
- **M（采购金额）**：单次¥500+→高价值店主，推荐的好友也可能高价值

```
推荐人价值评分 = R权重(0.3) + F权重(0.4) + M权重(0.3)
评分Top 10% → 优先邀请进入裂变活动
评分Bottom 30% → 暂不投入裂变资源
```

### 能力3：复盘ROI，优化投入产出

#### 🧠 底层逻辑

ROI不是"花了多少钱/拉了多少人"这么简单。B端裂变要算**全生命周期账**。

#### 🔧 操作思路

**A. 裂变ROI计算体系（3层递进）：**

```
Layer 1：短期ROI（活动级）
  ROI = 活动期间新增GMV - 活动总成本
        ──────────────────────────
                活动总成本
  用途：判断活动本身是否跑正

Layer 2：中期ROI（用户级）
  CAC（获客成本）= 总活动成本 / 新店主数
  LTV（生命周期价值）= 年均采购额 × 平均留存年限 × 2%（平台抽成）
  CAC回收期 = CAC / 月均GMV贡献
  用途：判断拉新的"质量"而不是"数量"

Layer 3：长期ROI（网络级）
  裂变系数 = 每个新店主在3个月内带来的额外新店主
  LTV×裂变系数 = 一个推荐人的"全链路价值"
  用途：判断推荐人的真实价值（高于CAC本身）
```

**B. 数据看板设计（你擅长的）：**

```
╔═══════════════════════════════════════════════╗
║             🔄 裂变活动实时看板               ║
╠═══════════════════════════════════════════════╣
║ 今日拉新: 47人  │ 本周: 312人  │ 本月: 1,246人 ║
║───────────────────────────────────────────────║
║ 推荐人排行                                    ║
║ 1. 广州-陈姐  推荐12人 首单转化率67%         ║
║ 2. 四季青-老王  推荐8人   首单转化率75%         ║
║ 3. 沙河-小林   推荐6人   首单转化率50%         ║
║───────────────────────────────────────────────║
║ 整体CAC: ¥63/人   │ LTV预估: ¥480/人          ║
║ CAC回收期: 4.2月  │ ROI: 1:7.6               ║
╚═══════════════════════════════════════════════╝
```

**C. 异常诊断三板斧：**

| 症状 | 可能原因 | 数据验证 | 我怎么做 |
|:-----|:---------|:---------|:--------|
| 曝光高→点击低 | 海报/推荐语不够吸引 | 改3版文案A/B测试 | 学你在硕牛的A/B测试方法 |
| 点击→注册低 | 注册流程太长/卡顿 | 看每一步的流失率 | 优化注册到一步完成 |
| 注册→首单低 | 新用户引导不够/信任不足 | 看7日内复访率 | 加新手任务+首单保障 |
| 某批推荐人效果下降 | 奖励疲劳/库存耗尽 | 看参与时长中位数 | 换激励方式/换话题 |

### 能力4：监控并规避活动风险（羊毛党）

#### 🧠 底层逻辑

B端的羊毛党比C端少，但一旦出现，**影响比C端大10倍**——因为B端活动奖励更高（¥30-50/人），而且虚假店主会污染数据、浪费运营资源。

#### 🔧 操作思路

**A. B端羊毛党的三种形态：**

| 类型 | 手段 | 识别特征 | 拦截手段 |
|:-----|:------|:---------|:---------|
| **设备批量** | 模拟器+批量手机号注册 | 注册IP集中、设备指纹重复 | 设备指纹+IP限流+手机号频次 |
| **关系造假** | 自己注册小号当"被推荐人" | 同一设备/网络注册的推荐关系 | 推荐人与被推荐人需不同设备、不同IP |
| **交易造假** | 小号刷首单→骗取推荐奖励 | 首单后退款率高、下单商品异常 | 首单奖励延后到"确认收货+7天无退货"后发放 |

**B. 风控架构（三层防御）：**

```
Layer 1：规则层（自动拦截90%的初级羊毛）
  ├ 同IP地址7天内关联≤3个注册
  ├ 推荐人-被推荐人非同一手机型号
  ├ 新注册账号24小时内不能作为推荐人
  └ 单日单设备最多邀请3人

Layer 2：算法层（识别复杂关系造假）
  ├ 社交图谱分析：检测"扇形"邀请结构（一个推荐人→多个新账号→无后续交易）
  ├ 行为轨迹分析：注册后的路径是否和正常店主一致
  └ 聚簇检测：一批账号是否有共同特征（注册时间集中/行为高度一致）

Layer 3：人工层（处理异常case）
  ├ 触发规则后自动标记为"观察期"——奖励冻结7天
  ├ 运营每周review一次异常账号池
  └ 确认羊毛后：取消奖励+封号+公示（起到震慑作用）
```

**C. 正向设计：让羊毛党"不值得薅"**

最好的风控不是"抓到了罚"，而是**设计上让羊毛党的收益远低于成本**：

```
奖励后置：不是"注册即得"，是"被推荐人首单后推荐人得"
  → 羊毛党需要真的下单 → 成本变高

时效限制：邀请链接7天有效
  → 批量注册的账号来不及在7天内都被推荐

金额阶梯：推荐第1人得¥30，第2人¥40，第3人¥50…
  → 批量小号的单次推荐收益低，不值得

反作弊隐藏：不公开活动规则中的风控细节
  → 羊毛党无法针对性破解
```

## 四、3个裂变活动模板（即拿即用SOP）

> 不管具体的活动主题怎么变，B端裂变活动的底层骨架只有3种。下面每个模板给一个**标准SOP** + **可替换变量** + **落地检查清单**。

---

### 模板A：信任裂变 — 「老带新·同行推荐」

**适用场景**：让已有店主把同行朋友带来平台注册/下单
**核心逻辑**：降低推荐者的信任风险 + 给被推荐者足够的保障
**驱动引擎**：人情+利益（但人情在前，利益在后）

#### SOP（标准执行流程）

```
Phase 1：筛选推荐人（3天）
  ├ 条件：月采购≥3次 + 社群活跃 + 朋友圈有选款内容
  ├ 人数：首批300-500人（宁可少，不要滥）
  └ 关键：挑选"同行认可"的店主，不是"最活跃"的店主

Phase 2：设计保障机制（核心差异化）
  ├ 推荐人："朋友首单后你得¥30-50现金"
  ├ 被推荐人："首单不满意平台包退邮费" + "首单立减¥20"
  └ 关键：首单保障是被推荐人的"安全网"也是推荐人的"面子保险"

Phase 3：发放推荐工具（1天）
  ├ 每位推荐官1个专属邀请码/专属海报
  ├ 海报要素：推荐官昵称+头像+"我推荐这家拿货" + 扫码注册
  └ 关键：海报看起来像"店主在推荐好货源"，不是"广告"

Phase 4：活动上线（持续追踪）
  ├ 推荐官把海报发朋友圈或私聊发给同行朋友
  ├ 朋友扫码→注册→填写推荐官邀请码
  ├ 朋友完成首单→推荐官¥30现金到账
  └ 关键：现金必须即时到账（微信/支付宝），不要拖

Phase 5：复盘迭代（第2周起）
  ├ 看数据：推荐官参与率、被推荐人注册率、首单转化率
  ├ 调优：改海报文案/调奖励金额/换推荐官
  └ 关键：每周迭代一个变量
```

#### 可替换变量

| 变量 | 选项A（保守） | 选项B（激进） | 何时选B |
|:-----|:------------|:------------|:--------|
| 推荐人奖励 | ¥30/人 | ¥50/人 | 首期转化率低于预期时 |
| 被推荐人奖励 | ¥10首单券 | ¥20首单券+包邮 | 竞品也在拉新时 |
| 门槛 | 被推荐人完成首单 | 被推荐人注册即可 | 急需冲注册量时 |
| 推荐人筛选 | RFM Top 5% | RFM Top 10% | 首批跑通需要放量时 |

#### 落地检查清单

```
□ 推荐奖励是否"被推荐人真实首单后"才发放？（防羊毛）
□ 推荐海报是否像"店主推荐"不像"广告"？
□ 被推荐人是否有首单保障？（信任安全感）
□ 推荐人奖励是否即时到账？（正反馈）
□ 推荐关系是否可解除？（B端必须给退出权）
□ 是否有推荐人行为监控？（防止过度推荐伤信任）
```

---

### 模板B：利益裂变 — 「拼单联盟·采购降价」

**适用场景**：让店主以"凑单"为理由拉同行一起采购
**核心逻辑**：省钱是B端最直接的利益驱动，拼单是合法的量价挂钩
**驱动引擎**：利益100%（不掺和人情，适合不熟但想拉人的场景）

#### SOP

```
Phase 1：选品（3天）
  ├ 选出5-10款"打包价有吸引力"的爆款
  ├ 设置拼单门槛：3件起/5件起/10件起（不同折扣）
  └ 关键：拼单折扣比单独买低5-15%

Phase 2：发起拼单（上线即生效）
  ├ 店主看到爆款→想拿但不够打包量→点击"发起拼单"
  ├ 系统生成拼单链接→店主发到同行群/朋友圈
  └ 关键：拼单链接打开直接显示"已有X人参与，还差Y件"

Phase 3：拼单完成（闭环）
  ├ 拼满→系统自动通知所有参与者→各自付款→各自发货
  ├ 拼不满→X小时后自动取消→不影响单独购买
  └ 关键：不扣款直到拼单成功，消除参与顾虑

Phase 4：裂变放大（社交设计）
  ├ "你拉了1个朋友和你拼→你俩都享受拼单价"
  ├ "你拉了3个朋友→你额外获得¥20无门槛券"
  └ 关键：拉人直接=省钱，不需要额外解释
```

#### 合规红线（必读）

```
《禁止传销条例》第七条 → 对照自查：
  ✅ 拼单折扣与"采购量"挂钩，不是"拉人头数"挂钩
  ✅ 非拼单用户可正常采购（只是不享受拼单价）
  ✅ 拼单失败自动取消，不影响正常下单
  ✅ 拼单关系可随时退出
  ✅ 没有"下线/上线"概念，只有"凑单"概念
  ✅ 参考拼多多拼团(2021)最高法知民终2112号：合法商业模式
```

#### 落地检查清单

```
□ 拼单折扣是否与采购量挂钩，不是人头数？（合规底线）
□ 不参与拼单的用户是否能正常下单？
□ 拼单失败是否自动退款/取消？
□ 拼单人数门槛是否≤10人？（门槛太高=没人参与）
□ 发起拼单的操作是否≤3步？
□ 是否有"帮拉人"一键分享功能？
```

### 模板C：内容裂变 — 「选款话题·晒单传播」

**适用场景**：让店主通过"晒选款"的方式自然传播品牌
**核心逻辑**：店主展示专业度+平台获得裂变素材+同行看到内容→自然转化
**驱动引擎**：荣誉+表达欲（利益是bonus）

#### SOP

```
Phase 1：定话题（每周/每两周一期）
  ├ 话题原则：低创作成本（3张图+50字）、高表达欲、可跟发
  ├ 话题示例：
  │  #这款我卖爆了  → 炫耀驱动力
  │  #踩雷款避坑    → 情绪共鸣驱动力  
  │  #今日探款      → 记录驱动力
  └ 关键：话题名必须是店主"本来就想说"的话

Phase 2：AI辅助生产（核心差异——0门槛参与）
  ├ 店主拍照上传→AI自动生成选款海报
  ├ AI自动写推荐文案（店主可改）
  ├ AI自动添加款式搜索编码
  └ 关键：店主操作<2分钟，不需要任何设计能力

Phase 3：内容发布+双渠道传播
  ├ 内容发布到平台"话题广场"→被其他店主看到
  ├ 同时生成分享版→店主可发朋友圈/社群
  └ 关键：分享版海报看起来是"选款推荐"不是"广告"

Phase 4：数据反馈（驱动持续参与）
  ├ 店主看到：你的内容被看了XX次、XX人通过你的推荐下单
  ├ 系统推送：恭喜！你的选款眼光被XX个同行认可
  └ 关键：数据反馈让店主获得"专业成就感"

Phase 5：话题TOP榜
  ├ 每周发布话题TOP10榜单→上榜店主获得官方推荐位
  ├ 连续3周上榜→认证为"选款达人"
  └ 关键：荣誉激励>现金激励
```

### 5.4 街拍模式的B端裂变SOP

```
Phase 1：「蹲点」发现"被埋没的选款眼光"（持续）
  ├ 运营每天在话题广场/晒单区/社群中浏览
  ├ 筛选标准：选款眼光好（款式好看/搭配合理）+ 内容呈现差（不会拍照/不会写文案）
  ├ 这类店主是"未被加工的内容宝石"
  └ 关键词：不是找最好的，是找"眼光好但不会表达"的

Phase 2：「主动邀请」免费帮加工（1对1）
  ├ 话术：
  │ "XX姐，我注意到你在平台上拿货眼光很好。
  │ 我们团队最近在做'店主选款推荐'项目，免费帮你看中的款
  │ 做成专业选款海报——不做任何商业使用。
  │ 做好之后图全部给你，你发朋友圈也行、自己存着也行。
  │ 唯一就是想问：如果做得好的话，能不能也发布到我们的
  │ 话题首页，让更多同行看到你的眼光？"
  ├ 关键：和抖音街拍一样——先免费提供服务，再征求同意
  └ 和抖音街拍一模一样的逻辑：
      "免费帮你修图" = "免费帮你做专业选款海报"
      "我能发到抖音吗？" = "我能发到话题首页吗？"

Phase 3：「内容加工」AI+人工双重加持
  ├ AI生成海报初稿（排版+文案+搜索编码）
  ├ 运营人工微调（保证质量，加入"推荐官"印记）
  ├ 交付给店主：海报原图+不带水印版
  └ 关键：海报质量必须让店主"有面子发朋友圈"

Phase 4：「发布+传播」双赢
  ├（经店主同意后）发布到话题首页
  ├ 店主自己发朋友圈（内容足够好→店主有分享动力）
  ├ 其他同行看到→"这个款我也拿过"→参与话题
  └ 飞轮：更多优质内容→更多曝光→更多店主想被"加工"

Phase 5：「放大」从"平台加工"到"KOC互拍"
  ├ 当模式跑通后：不仅是平台帮店主加工内容
  ├ 可以鼓励"资深店主KOC"去发现和帮"新店主"加工
  ├ 资深店主获得了行业影响力+平台推荐奖励
  ├ 新店主获得了专业内容+被认可感
  └ 和抖音街拍生态一模一样：摄影师越来越多，素人越来越多
```

### 5.5 为什么这个模式在B端特别有效——5个原因

| 原因 | 解释 |
|:-----|:------|
| **① 解决了"推荐风险"** | 店主不需要主动"拉人"，他只是允许平台展示他的选款——推荐风险从店主转移到了平台 |
| **② 满足了"专业自尊心"** | 服装店主最引以为豪的是"我选款眼光好"——这个模式恰恰是肯定并且放大了这种自豪感 |
| **③ 零广告感** | 和抖音街拍一样，内容看起来是"一个真店主的选款推荐"，而不是"平台的广告" |
| **④ 激励机制自然** | 店主不需要为了钱去分享（像传统裂变），而是因为"我的眼光被认可了"去分享 |
| **⑤ 飞轮自转** | 优质内容→更多曝光→更多店主想被加工→更多优质内容→循环 |

### 5.6 面试时讲这个（加分项）

> "我最近观察到抖音的一个趋势——素人街拍。摄影师在东山口这些地方帮路人免费拍照，修好图免费给人家，然后问'我能发到抖音吗？'——大部分人同意了，于是就有了那些爆款街拍视频。
>
> 我觉得这个模式翻译到一手的B端裂变特别有意思：**把"让店主拉人"变成"帮店主变得更好，然后他自愿让你展示"。**
>
> 具体就是：运营每天在话题区发现那些选款眼光好但不会拍照写文案的店主——免费帮他们做专业选款海报，海报做得足够好，店主愿意发朋友圈。这时候裂变不是"任务"，是"顺带的事"——同行看到他的选款→对一手产生信任→注册/下单。
>
> 这个模式的精妙之处和抖音街拍一样：**你先是帮了用户一个忙，然后他才同意让你用他的内容。而不是你让用户帮你一个忙（拉人），然后给他钱。前者的内容是真实的、有信任感的；后者的海报是广告味的、没人看的。**"

## 六、从0搭建积分体系（与裂变互补的长期基建）

> **🔍 现状（已通过APP截图验证）**：已确认一手APP个人中心截图，**没有积分体系、没有会员等级、没有签到、没有积分商城**。现有资产仅"钱包/卡券/红包"三项基础功能。具体分析见[6.10 截图拆解](#610-截图拆解一手app个人中心-用户运营视角)。
> 
> 这意味着：
> - 裂变活动目前只能靠**一次性现金奖励**驱动 → 成本高、持续性差
> - 店主在平台上**没有"积累感"** → 留存靠采购习惯，而非体系绑定
> - 但反过来，**这正是你的机会**——你有希尔顿会员体系0→1的实战经验，这个需求完全可以做成面试时的加分方案
>
> **核心定位**：积分体系 ≠ 裂变活动的替代品，而是让裂变效果**被锁定、被放大**的基建。

### 6.1 底层逻辑：B端积分体系 ≠ C端积分体系

| 维度 | C端积分（如淘宝淘金币） | B端积分（一手的版本） |
|:-----|:---------------------|:-------------------|
| **核心目的** | 提高复购率、增加活跃 | **降低采购成本、锁定采购渠道** |
| **用户动机** | "攒积分兑换商品" | "积分直接抵货款=进货更便宜" |
| **积分价值感** | 低（100积分≈1元） | 必须高（10积分≈1元，甚至更高） |
| **核心权益** | 兑换实物/优惠券 | **货款抵扣、优先选款、包邮、爆款配额** |
| **等级标准** | 消费金额 | **采购额+采购频次+裂变贡献** |
| **积分获取** | 签到/购物/玩游戏 | **采购返积分+裂变奖励积分+内容贡献积分** |

**B端积分体系的核心公式：**

```
店主在平台上的"总价值" = 采购省下的钱 + 积分兑换的权益 + 裂变赚到的奖励
                                                      │
                                        积分体系在这里放大这个值
                                          让每一笔采购都"攒下未来"
```

**你从希尔顿会员体系经验可以迁移的：**
> 你在万富做过希尔顿4级会员体系0→1——等级划分、门槛设计、权益配置、数据验证（高级会员10%人数贡献34%GMV）。这个经验放在一手就是：**把"店主在平台上的采购行为"转化成"可积累的资产"**——采购越多、等级越高、权益越大、裂变贡献越强。

### 6.2 积分体系五层架构

                    │   第五层：等级体系    │
                    │  青铜→白银→黄金→铂金  │
                    │  等级越高权益越大     │
                    └─────────┬───────────┘
                              │
                    ┌─────────┴───────────┐
                    │   第四层：权益中心    │
                    │  货款抵扣·包邮·优选权  │
                    │  专属客服·线下活动    │
                    └─────────┬───────────┘
                              │
                    ┌─────────┴───────────┐
                    │   第三层：积分消耗    │
                    │  怎么花——花在最有价值的地方 │
                    └─────────┬───────────┘
                              │
                    ┌─────────┴───────────┐

                              │
                    ┌─────────┴───────────┐

```

### 6.3 第一层：底层数据与合规（先搭地基）

**积分的经济学模型（必须在第一层确定）：**

```
积分发行量 = 预算成本 ÷ 积分面值
            ↓
假设：每月预算¥50,000做积分激励
     → 每100积分=¥1货款抵扣
     → 每月可发行500万积分
     → 单个店主月均最多赚2000积分（¥20抵扣）
     → 防止积分泛滥导致"通货膨胀"
```

**合规底线（务必和法务确认）：**
- ✅ 积分**不能直接兑换现金**（否则涉及金融监管）
- ✅ 积分**有有效期**（避免长期负债）
- ✅ 积分**不是投资品**（不可转让/交易）
- ✅ 积分获取行为**可追溯**（防刷）
- ❌ 积分不能和"拉人头"直接挂钩（传销红线）

### 6.4 第二层：积分获取（引导店主做平台想要的行为）

| 行为类型 | 具体行为 | 积分奖励 | 频次上限 | 设计目的 |
|:--------|:---------|:--------|:--------|:---------|
| **🛒 采购行为**（核心） | 每完成一单采购 | 订单金额×1%积分 | 无上限 | 核心采购激励，积分=变相返利 |
| **📝 内容行为** | 发布选款晒单内容 | 50积分/条 | 3条/天 | 增加内容供给，为内容裂变供料 |
| **🗣️ 裂变行为** | 推荐同行注册→首单 | 200积分/人 | 10人/周 | 裂变驱动力（推荐奖励+积分双激励） |
| **👋 日常行为** | 每日签到 | 5积分/天 | 1次/天 | 养成打开APP习惯 |
| **🏆 成就行为** | 话题TOP10上榜 | 500积分/次 | 月结 | 激励内容质量 |

**设计逻辑：**
- 采购行为是"主干"——积分和采购额挂钩，让店主觉得"每笔采购都在攒钱"
- 裂变行为是"放大器"——积分+现金双激励，降低推荐的心理门槛
- 日常行为是"钩子"——把签到和内容发布变成日常动作

### 6.5 第三层：积分消耗（让积分"值钱"才有意义）

B端店主最在乎什么？**进货省钱**。所以积分的核心消耗场景必须围绕"降低采购成本"：

| 消耗场景 | 兑换比例 | 用户感知价值 | 优先级 |
|:---------|:--------|:------------|:------|
| **🥇 货款抵扣**（核心） | 100积分=¥1 | ⭐⭐⭐⭐⭐ | **必须第一个上** |
| **🥈 快递费抵扣** | 200积分=包邮1单 | ⭐⭐⭐⭐ | 第二优先级 |
| **🥉 爆款优先购买权** | 500积分=优先购1款 | ⭐⭐⭐ | 高等级用户专属 |
| **🎫 线下活动入场券** | 1000积分=1次 | ⭐⭐⭐ | 增加社区粘性 |
| **🎁 限定款抽签资格** | 200积分=1次抽签 | ⭐⭐ | 趣味性补充 |

**关键设计原则：**
> 积分抵扣货款是**必须有的核心场景**——如果积分不能帮店主省钱，这个体系就没有存在的意义。其他所有消耗场景都是锦上添花。

### 6.6 第四层：等级体系（用"身份"锁定高价值店主）

借鉴你在希尔顿做的4级会员体系：

```

#### 🔍 用户运营视角的7个关键发现

**发现1：已存在裂变入口「邀请有奖」——但非常初级**
> "邀请有奖"在活动区底部，文案"返8-100元券包"即推荐人获得的是**优惠券**不是现金，这对B端店主吸引力有限。且没有区分"推荐人"和"被推荐人"的双边激励，也没有推荐关系追踪的可见入口。
>
> **→ 升级方向**：在现有"邀请有奖"基础上叠加推荐关系绑定+现金奖励+积分奖励，而不是从0做新入口。

**发现2：「分销管理」入口已存在——说明分销体系有基础**
> 四个数据入口中有"分销管理"，说明已有或曾经有过分销相关功能。但入口位置在"订阅档口/收藏关注/足迹"同一层级，属于**边缘功能位**。
>
> **→ 升级方向**：把分销管理提升为独立模块，搭配推荐裂变活动一起做。

**发现3：「集财运」活动——唯一有"积分感"的入口**
> "集财运"是当前App内**唯一接近积分/游戏化**的功能。但从入口文案"赚财运享福利"看，更像是一个**活动型功能**（限时），不是持久的积分体系。
>
> **→ 策略**：调研"集财运"的活动数据——如果参与率高，说明店主对"攒→兑"机制有需求；如果参与率低，说明需要更直接的价值驱动。

**发现4：资产模块极度薄弱——只有钱包/卡券/红包**
> 这是最核心的发现。一个年GMV 30亿+的平台，**个人资产只有三项基础功能**：
> - 钱包（余额）→ 可能只是退款/提现用
> - 卡券（优惠券）→ 基础营销工具
> - 红包（现金红包）→ 临时激励
>
> **没有任何"积累型资产"**——没有积分、没有成长值、没有经验值、没有会员等级。
>
> **→ 这恰好是你方案的核心价值点**：把"我的资产"从3项扩充到4项（+积分），让店主有累积感。

**发现5：店主认证是"一次性开关"不是"成长体系"**
> 认证店主提供的权益（批发价/无理由退货/运费券/新款抢先看）是**统一的一刀切权益**，认证后所有人一样。没有"青铜→白银→黄金→铂金"的等级差异。
>
> **→ 升级方向**：在"一键认证"的基础上叠加等级体系——认证只是门槛，等级才是差异化权益。

**发现6：没有签到入口——日活养成缺失**
> 个人中心上下两页均无签到入口。这意味着平台目前**没有通过"每日动作"培养用户打开习惯**。
>
> **→ 升级方向**：积分体系的第一步就可以是签到得积分——低成本、高养成价值。

**发现7：活动区有5个活动——但没有"常设积分商城"**
> 新客首单权益、发现宝藏档口、店主福利社群、集财运、邀请有奖——都是**限时/一次性活动**。没有积分商城这类**常设的价值兑换出口**。
>
> **→ 升级方向**：在活动区增加一个常驻的"积分商城"入口，让积分有地方花。

#### ⛔ 谦腾模式的3个致命风险（必须自己先说）

```
风险1：合规风险 — 涉及二级分销（平台→大团长→小团长）

风险2：粉丝反噬 — 信任的不可逆损伤

风险3：不可持续 — 裂变快≠模式好
  → 增长靠"拉人头"驱动，不是靠产品价值驱动
  → 一旦增长见顶（近100万），留存和活跃才是真问题

#### 🧠 真正可以迁移到B端的3个思维

| 谦腾经验中的"思维" | 不能直接搬的部分 | 可以B端化的翻译 |
|:------------------|:---------------|:--------------|
| **分层管理** — 平台只管顶层1000个大团长，下层自动繁殖 | 多级分佣体系（涉及二级） | 只管理TOP推荐官，但推荐关系**只到一级**——推荐人和被推荐人，不再往下延伸 |
| **利益绑定** — 大团长的收入取决于团队产出，所以自发动力 | 层层分佣的裂变结构 | 一次性推荐奖励+被推荐人持续采购的积分返还——**只奖直接推荐，不奖间接推荐** |
| **自发带教** — 大团长主动教小团长，因为教好了自己赚更多 | 上下线关系（有控制意味） | 推荐人帮被推荐人上手→双方各得奖励（推荐人得积分，被推荐人更快拿到首单返利）——**鼓励"帮带"，但不锁定关系** |

#### 改造后的B端安全版：「店主推荐积分计划」

| 要素 | 谦腾原版 | B端安全版 |
|:-----|:---------|:----------|
| **层级** | 平台→大团长→小团长→群成员（3层） | 老店主→新店主（**仅1层**） |
| **推荐奖励** | 下级的消费分佣（持续抽成） | 一次性推荐奖金+被推荐人首年采购额1%积分（**积分可抵货款，非现金抽成**） |
| **关系锁定** | 上下线锁定，不可退出 | 推荐关系自然绑定，但**任何一方可主动解除** |
| **平台管理** | 管理大团长（间接管所有层级） | 管理所有推荐官**但只看直接推荐数据** |
| **风控底线** | 依赖外部平台（微信）容忍度 | **自建体系，全链路可控** |
| **合规依据** | 灰色地带 | 对照《禁止传销条例》第七条逐一自查，确保不构成"拉人头" |

**执行路径：**

```
Phase 1：筛选种子推荐官（基于RFM）
  → 选月采购≥3次+社群活跃的店主作为首批推荐官
  → 和谦腾一样：只管顶层，但推荐只到一层

Phase 2：设计激励（一次性+持续性）
  一次性：新店主首单后，推荐人得¥30-50现金
  持续性：新店主首年采购额1%转化为推荐人积分（可抵货款）
  关键差异：没有"间接推荐"（你的下线的下线和我没关系）

Phase 3：帮带机制（"大团长带小团长"思维的B端版本）
  推荐人帮新店主解决前3次采购的选款/下单问题
  → 新店主更早上手→更早出首单→推荐人更早拿到奖励
  不需要"上下级绑定"，是"帮你有好处→自然愿意帮"

Phase 4：退出机制（避免谦腾的"粉丝反噬"）
  推荐关系可解除：任何一方不满→主动解除推荐关系
  被推荐人不会"被锁定"在被抽成的体系里
  → 这是B端和C端分销的核心区别：B端一定要给退出权
```
